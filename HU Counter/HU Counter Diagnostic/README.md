# HU Counter Diagnostic (Edge/Chrome MV3)

Диагностическая версия расширения HU Counter для проверки:
- распознавания целевой SAP-страницы разными способами;
- поиска HU разными стратегиями;
- стабильности в SPA/динамической загрузке;
- производительности методов (время, объем текста, число узлов).

## Что внутри
- `manifest.json` — MV3-манифест с `scripting`, `storage`, `activeTab`.
- `src/content.js` — все диагностические методы + toast + `console.table` + отправка истории в storage.
- `src/background.js` — сохранение/выдача истории, отдельный метод через `chrome.scripting.executeScript`.
- `src/toast.css` — стиль toast-уведомлений.
- `popup/popup.html`, `popup/popup.js` — просмотр последних запусков диагностики.
- Иконки намеренно не добавлены, чтобы сборка не содержала бинарные файлы PNG (совместимость с текущим review-процессом).

## Загрузка в Edge
1. Открыть `edge://extensions`.
2. Включить `Developer mode`.
3. Нажать `Load unpacked`.
4. Выбрать папку `HU Counter/HU Counter Diagnostic`.

## Как тестировать
1. Открыть целевую SAP-страницу.
2. Дождаться серии toast-уведомлений справа сверху.
3. Открыть DevTools Console и проверить `console.table()` с результатами.
4. Открыть popup расширения и проверить сохраненную историю результатов.

## Критерии выбора рабочего метода
Рабочим считать метод, который:
1. стабильно находит HU на рабочем ПК;
2. не зависит от одного конкретного timing-сценария;
3. имеет приемлемое время выполнения;
4. не сканирует чрезмерно большой объем DOM без необходимости.

## Рекомендация для переноса обратно
После диагностики перенести в основной `HU Counter`:
- лучший метод распознавания страницы (обычно URL + DOM signature + fallback через observer);
- лучший метод поиска HU (точный regex + fallback regex + селекторы);
- безопасный fallback по таймингам (например, 500/1500ms и MutationObserver с ограничениями).
